# cdlg — Console Dialogs для Claude Code

> [!NOTE]
> [Read in English](README.md)

Минималистичный браузер сессий для [Claude Code](https://claude.ai/code). Откройте любой прошлый диалог, посмотрите статистику токенов или начните новый — всё из одного меню.

## Возможности

- **Статистика токенов** — ввод, кэш-запись, кэш-чтение, вывод одним взглядом
- **Возобновление по номеру** — никаких UUID, никакого копания в `~/.claude/projects/`
- **Двуязычный интерфейс** — RU/EN, автоопределение по `$LANG`, переключение клавишей `l`
- **Нет зависимостей** — bash + python3 stdlib, ничего устанавливать не нужно
- **Без sudo** — устанавливается в `~/.local/bin` через `--install`
- **bash 3.2+** — работает на macOS из коробки

## Концепция

Claude Code уже является консольным чатом. `cdlg` — это навигационный слой поверх него: читает `~/.claude/projects/` и показывает диалоги в виде нумерованного списка со статистикой токенов. Никаких дополнительных процессов, никакого сервера — только меню, которое передаёт управление `claude`.

Предполагаемая схема использования — **отдельная директория для диалогов**:

```
~/dialogs/          ← здесь хранятся ваши личные разговоры с Claude
```

Все сессии, запущенные из этой директории, группируются вместе и отображаются в `cdlg`.

## Требования

- [Claude Code](https://claude.ai/code) установлен и авторизован (`claude` в PATH)
- `bash` 3.2+
- `python3` (стандартный, без дополнительных пакетов)

## Установка

**Быстрая установка (рекомендуется)**

```bash
curl -fsSL https://raw.githubusercontent.com/AliceWork86/cdlg/v0.0.1/cdlg.sh -o /tmp/cdlg.sh
bash /tmp/cdlg.sh --install
```

Устанавливает в `~/.local/bin/cdlg`. `sudo` не нужен. Если `~/.local/bin` не в PATH — скрипт выведет нужную строку для `~/.bashrc` / `~/.zshrc`.

**Из исходников**

```bash
git clone https://github.com/AliceWork86/cdlg.git
bash cdlg/cdlg.sh --install
```

**Запуск на месте**

```bash
git clone https://github.com/AliceWork86/cdlg.git
bash cdlg/cdlg.sh
```

Запускает `cdlg` из текущей директории — установка не нужна.

## Первый запуск

`cdlg` показывает сессии директории, из которой запущен — так же, как Claude Code их группирует.

**Чтобы увидеть существующие сессии**, запусти `cdlg` из папки, где обычно работаешь с Claude:

```bash
cd ~/my-project
cdlg
```

**Чтобы хранить все личные диалоги в одном месте**, выбери отдельную папку:

```bash
mkdir -p ~/dialogs
cd ~/dialogs && cdlg
```

С этого момента всегда запускай `cdlg` из `~/dialogs` — или задай `CDLG_DIR`, чтобы работало откуда угодно:

```bash
export CDLG_DIR="$HOME/dialogs"  # добавь в ~/.bashrc или ~/.zshrc
cdlg
```

## Использование

| Клавиша | Действие |
|---------|----------|
| `1`–`N` | Возобновить диалог N |
| Enter | Начать новый диалог |
| `l` | Переключить язык (RU ↔ EN) |
| `q` | Выход |

## Настройка

Отредактируйте две переменные в начале `cdlg.sh`:

```bash
CDLG_DIR=""    # Абсолютный путь к директории диалогов.
               # Пусто = текущая рабочая директория.
CDLG_LANG=""   # Язык: ru, en. Пусто = автоопределение по $LANG.
```

Или задайте через переменные окружения, не трогая файл:

```bash
export CDLG_DIR="$HOME/dialogs"
export CDLG_LANG="ru"
```

## Колонки токенов

| Колонка | Значение |
|---------|----------|
| `↑ ввод` | Реальные входные токены (тарифицируются) |
| `+ кэш-запись` | Токены, записанные в кэш промпта |
| `~ кэш-чтение` | Токены из кэша (дешевле) |
| `↓ вывод` | Сгенерированные токены ответа |

Значения ≥ 1000 отображаются как `Xk`. Все значения накопительные за всю сессию.
